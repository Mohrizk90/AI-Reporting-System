# Demo checklist — Pedicel Paid Ads MVP (Meta)

> Note: This repo now also includes an **Email (Brevo)** module. Paid Ads and Email are demoed separately.

## Prerequisites

- Python 3.10+ with `pip install -r requirements.txt` (ensures **`openai>=1.40.0`** — required for current `httpx`)
- Optional: `.env` in the project root with `OPENAI_API_KEY=` for LLM-generated narratives (copy from `.env.example`). **Do not commit `.env`.**
- For Email (Brevo): set `BREVO_API_KEY=` in `.env`. If you pasted an API key in chat, **rotate it**.
- If you have an “MCP API” value that looks like base64 (e.g. `eyJ...==`), **do not** use it directly as `BREVO_API_KEY`. Extract the raw `xkeysib-...` key and use that instead.

### Confirm the API key path works

1. Run **without** `--no-ai` (same command as “With AI” below).
2. Open `report.json` → **`insights.source`** should be **`openai`** (not `stub` or `error`).
3. §5.1 body should be a real narrative, not the stub text.

If you see **`Client.__init__() got an unexpected keyword argument 'proxies'`** in `insights.caveats`, upgrade the SDK: `python -m pip install --upgrade "openai>=1.40.0"`.

## Outputs

Default directory: **`output/<client_id>/`**

| File | Purpose |
|------|---------|
| `report.json` | Full tree (`paid_ads_report`, `scope`, `insights`) |
| `report.txt` | Plain-text client-style layout |
| `report.md` | Markdown |
| `report.html` | **Print-friendly** HTML for demo / internal review |

Override: `--output-dir path\to\folder`

## What is implemented

- Meta campaign export → §1 (incl. executive summary), §2.1–2.3, §4 reference benchmarks, §5 (stub or LLM)
- Explicit **scope** block: supported vs out-of-scope
- **Not implemented (Paid Ads):** Google §3, §2.2 creative metrics, SEO/social/email (listed in JSON, not implied as live data)

### Important data note

- `Files/YOA Campaigns-1-Sep-2025-31-Mar-2026.xlsx` is a **Meta Ads export**, not an Email export. Email Excel ingestion is **pending** until a real ESP export is provided.

## What to show in a demo

1. Open **`report.html`** — walk §1 executive summary → §2.3 campaigns → §3 “not included” → §5.
2. Show **`scope`** in JSON for engineering transparency.
3. If presenting to clients, use **HTML or PDF (print from browser)**; keep raw JSON for internal use.
